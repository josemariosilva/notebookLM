# Miniguia de Estudos

## O que é Teste de Software?

É o processo de verificar e validar se um software atende aos requisitos funcionais e não funcionais, identificando defeitos antes da entrega ao usuário.

---

## Teste Manual

Características:

- Executado por pessoas
- Ideal para testes exploratórios
- Melhor percepção da experiência do usuário

---

## Teste Automatizado

Características:

- Executado por ferramentas
- Rápido
- Repetível
- Excelente para regressão

---

## Tipos de Testes

### Teste Unitário

Valida pequenas partes do código.

### Teste de Integração

Verifica comunicação entre módulos.

### Teste de Sistema

Avalia toda a aplicação.

### Teste de Aceitação

Valida requisitos de negócio.

### Teste Exploratório

Baseado na experiência do testador.

### Teste de Regressão

Garante que novas alterações não quebraram funcionalidades existentes.

---

## Cobertura de Código

Mede quanto do código foi executado pelos testes.

Importante lembrar:

Alta cobertura não significa ausência de defeitos.

---

## CI/CD

### Continuous Integration

Integra alterações frequentemente utilizando testes automatizados.

### Continuous Delivery

Mantém o software sempre pronto para produção.

### Continuous Deployment

Publica automaticamente versões aprovadas pelos testes.
